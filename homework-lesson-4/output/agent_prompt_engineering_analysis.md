# Аналіз практик Prompt Engineering у роботі дослідницького агента

## Короткий висновок
Цей звіт аналізує архітектуру та поведінку поточного дослідницького агента через призму технік Prompt Engineering, представлених у лекції. Агент не просто виконує запит, а демонструє складний **Agentic Workflow**, що поєднує кілька передових методів для досягнення високої точності та надійності результату.

## Виявлені практики Prompt Engineering

### 1. ReAct (Reasoning and Acting)
Це фундаментальна архітектура цього агента. Замість того, щоб давати відповідь на основі лише внутрішніх знань (що призводить до галюцинацій), агент використовує цикл:
- **Thought:** Планування кроків (пошук запитів).
- **Action:** Виклик інструментів (`web_search`, `read_url`).
- **Observation:** Аналіз отриманих даних.
Це дозволяє агенту коригувати свій шлях у реальному часі, що є ключовою ознакою **Agentic RAG**.

### 2. Agentic RAG (Retrieated Augmented Generation)
Агент не просто отримує контекст, він сам керує процесом ретривалу. Він вирішує:
- Які саме запити сформулювати для пошуку.
- Які URL є достатньо релевантними для читання.
- Коли інформації вже достатньо, щоб припинити пошук.
Це перехід від пасивного отримання контексту до активного дослідження.

### 3. Self-Reflection (Саморефлексія)
Перед фіналізацією звіту агент виконує етап критичного аналізу:
- Перевірка відповідності висновків знайденим джерелам.
- Оцінка повноти структури звіту.
- Виявлення та чесне зазначення обмежень (наприклад, якщо певна інформація не була знайдена).
Це запобігає генерації "впевнених, але помилкових" відповідей.

### 4. Few-shot / Instruction Following
Хоча в поточному запиті не було надано прикладів (Zero-shot), системний промпт містить чіткі інструкції щодо формату (Markdown, структура звіту) та правил безпеки. Агент демонструє здатність до суворого дотримання складних протоколів (наприклад, не показувати внутрішній `Thought` користувачу).

### 5. Chain-of-Thought (CoT)
Хоча процес мислення прихований, логіка агента побудована на послідовному розв'язанні підзадач: *Пошук $\rightarrow$ Читання $\rightarrow$ Синтез $\rightarrow$ Збереження*. Кожен наступний крок логічно випливає з результатів попереднього.

## Порівняльна таблиця застосованих технік

| Техніка | Роль в агенті | Результат для користувача |
| :--- | :--- | :--- |
| **ReAct** | Цикл прийняття рішень | Зменшення галюцинацій, актуальність даних |
| **Agentic RAG** | Керування інструментами пошуку | Глибоке дослідження теми, а не поверхневий огляд |
| **Self-Reflection** | Валідація результату | Висока достовірність та чесність щодо обмежень |
| **Structured Output** | Форматування Markdown | Зручність читання та професійний вигляд звіту |

## Обмеження
- Аналіз базується на спостереженні за логами виконання поточного сеансу.
- Повна оцінка "Self-consistency" (здатність агента давати однакові відповіді при різних прогонах) потребує проведення масштабних тестів (eval-driven workflow).
