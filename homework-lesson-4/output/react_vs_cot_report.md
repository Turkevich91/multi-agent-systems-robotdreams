# Дослідження ReAct loop та його порівняння з Chain-of-Thought (CoT)

## Висновок
ReAct (Reasoning and Acting) — це фреймворк, який поєднує логічне мислення (reasoning) з діями (acting) у єдиний цикл. На відміну від класичного Chain-of-Thought, ReAct дозволяє моделі взаємодіяти із зовнішнім світом через інструменти (пошук, калькуляції тощо), що робить її менш схильною до галюцинацій та більш адаптивною до нових даних.

## Аналіз: ReAct vs Chain-of-Thought

| Характеристика | Chain-of-Thought (CoT) | ReAct (Reasoning + Acting) |
| :--- | :--- | :--- |
| **Основний принцип** | Генерація послідовності логічних кроків для досягнення відповіді. | Почергова генерація міркувань та виконання конкретних дій (tools). |
| **Доступ до знань** | Обмежений лише внутрішньою базою знань моделі (training data). | Може отримувати актуальну інформацію з зовнішніх джерел (web search, API). |
| **Механізм роботи** | "Думати" -> "Відповідати". | "Думати" -> "Діяти" -> "Спостерігати" -> "Думати далі". |
| **Ризик галюцинацій** | Високий при запитах на факти, яких немає в навчальній вибірці. | Нижчий, оскільки модель може перевірити факти через інструменти. |
| **Складність** | Простіший у реалізації (просто промпт). | Складніший (потребує інтеграції з інструментами та обробки результатів). |

### Як працює ReAct loop
Процес складається з циклу:
1. **Thought (Міркування):** Модель формулює план або аналізує попередній крок.
2. **Action (Дія):** Вибір конкретного інструмента (наприклад, `web_search`) та формування запиту.
3. **Observation (Спостереження):** Отримання результату від зовнішнього середовища (текст зі сторінки, результат пошуку).
4. **Повторення:** На основі спостереження модель оновлює свій план і переходить до наступного кроку "Thought".

## Практичні trade-offs (Компроміси)

**Переваги ReAct:**
* **Актуальність:** Можливість працювати з подіями, що відбулися після завершення навчання моделі.
* **Інтерпретованість:** Користувач бачить кожен крок логіки та кожну дію, що дозволяє відстежити помилку.
* **Точність:** Зменшення ефекту "самопідтвердження" помилок через перевірку фактів.

**Недоліки ReAct:**
* **Затримка (Latency):** Кожен цикл потребує окремих викликів LLM та очікування відповіді від інструментів, що робить процес повільнішим.
* **Вартість (Cost):** Більша кількість токенів витрачається на повторні запити з контекстом попередніх кроків.
* **Залежність від якості інструментів:** Якщо пошукова система або API повертає шум, модель може заплутатися в логіці.

## Обмеження
* ReAct не є "магічною пігулкою": якщо модель не вміє правильно формулювати запити до інструментів (Action), цикл не принесе користі.
* При дуже складних завданнях кількість кроків може зростати експоненціально, що призводить до виходу за межі контекстного вікна.
