# Оптимальне налаштування векторних баз даних для агентних систем (Agentic Systems)

## Резюме (Executive Summary)
Для автономних агентних систем (agentic systems) векторна база даних є не просто сховищем, а ключовим компонентом **довготривалої пам'яті**. Оптимальне налаштування має фокусуватися на вирішенні трьох основних проблем: **мінімізація затримки (latency)** для швидких циклів роздумів, **підвищення точності (precision)** для уникнення галюцинацій та **ефективне управління контекстом** через стратегії стиснення пам'яті.

## 1. Архітектура пошуку та індексації
Для агентів критично важливо збалансувати швидкість доступу до "активної пам'яті" та здатність зберігати величезні обсяги "історичної пам'яті".

### Стратегії індексації
* **HNSW (Hierarchical Navigable Small World):** Найкращий вибір для **short-term memory**. Забезпечує надшвидкий пошук (low latency), необхідний під час активного циклу роздумів агента, але потребує багато оперативної пам'яті.
* **IVF (Inverted File Index):** Оптимальний для **long-term memory**. Дозволяє масштабувати базу до мільйонів записів, економлячи ресурси за рахунок кластеризації векторного простору.
* **Product Quantization (PQ):** Рекомендовано для стиснення векторів, що дозволяє агенту зберігати значно більший обсяг епізодичної пам'ятність у доступній RAM.

### Покращення точності: Re-ranking та Multi-vector Retrieval
Щоб агент не робив помилок на основі релевантних, але неточних фрагментів (noise), необхідно впроваджувати:
* **Re-ranking (Cross-encoders):** Другий етап пошуку, який перевіряє топ-$k$ результатів за допомогою важкої, але точної моделі. Це критично для зменшення галюцинацій.
* **Parent-Document Retrieval:** Пошук здійснюється за дрібними чанками (для високої точності), але агенту передається повний контекст батьківського документа (для розуміння суті).
* **Small-to-Big Retrieval:** Дозволяє агенту "зуммуватися" на конкретну деталь і автоматично розширювати вікно контексту для глибшого аналізу.

## 2. Гібридний пошук та метадані
Чистий семантичний пошук часто не справляється з технічними термінами або точними даними.

* **Hybrid Search (Dense + Sparse):** Поєднання векторного пошуку (embeddings) з класичним текстовим пошуком (BM25). Це необхідно для роботи з кодом, назвами артикулів та специфічною термінологією.
* **Metadata Filtering:** Використання атрибутів (`session_id`, `timestamp`, `importance_score`) дозволяє агенту спочатку звузити область пошуку (наприклад, "тільки за вчорашній день"), що радикально знижує рівень шуму та витрати на обчислення.

## 3. Управління пам'яттю агентів
Ефективний агент повинен самостійно керувати своїм станом:
* **Episodic Memory:** Збереження логів дій (action outcomes). Агент має бачити не лише текст, а й результат своєї дії (успіх/помилка).
* **Semantic Memory:** Збереження абстрактних знань та фактів.
* **Memory Summarization:** Процес "компресії", де агент періодично перетворює довгі діалоги на стислі резюме, записуючи їх у БД, щоб не перевищувати ліміт контекстного вікна (Context Window).

## 4. Порівняння векторних баз даних

| Рішення | Сильні сторони для агентів | Слабкі сторони | Найкраще використання |
| :--- | :--- | :--- | :--- |
| **Pinecone** | Managed (SaaS), швидкість, автоматичне масштабування. | Закритий код, висока вартість при великих обсягах. | Продакшн-агенти (Time-to-Market). |
| **Milvus** | Висока продуктивність на мільярдах векторів, гнучкість. | Складність розгортання та адміністрування. | Enterprise-системи з величезними даними. |
| **Weaviate** | Вбудована підтримка RAG модулів, чудовий гібридний пошук. | Високе споживання ресурсів при складних схемах. | Агенти з високою інтеграцією з LLM. |
| **Qdrant** | Швидкість (Rust), потужний фільтр за метаданими. | Менша екосистема порівняно з Milvers/Pinecone. | Високонавантажені сервіси, Edge-агенти. |
| **Chroma** | Максимальна простота, ідеально для прототипів. | Обмежене масштабування (Scale-out). | R&D, локальні персональні асистенти. |

## Висновки та рекомендації
Для побудови надійної агентної системи:
1. **Стандарт:** Використовуйте **HNSW + Metadata Filtering** для балансу швидкості та точності.
2. **Точність:** Обов'язково впроваджуйте **Re-ranking** (Cross-encoders) для фільтрації галюцинацій.
3. **Контекст:** Використовуйте стратегію **Parent-Document Retrieval**, щоб агент мав повну картину подій.
4. **Автономність:** Реалізуйте механізм **Summarization** для автоматичного очищення контексту.
